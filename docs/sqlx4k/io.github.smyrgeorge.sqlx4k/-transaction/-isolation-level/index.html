IsolationLevel

enum IsolationLevel : Enum<Transaction.IsolationLevel> (source)

Enum class defining the isolation levels for database transactions.

The isolation level determines how transaction integrity is maintained and how concurrent transactions interact with each other. Each level provides different guarantees regarding visibility of changes made by other transactions, and controls phenomena like dirty reads, non-repeatable reads, and phantom reads.

Isolation levels:

  • ReadUncommitted: Allows transactions to read data modified by other transactions that have not yet been committed.

  • ReadCommitted: Ensures that only committed data is read, preventing dirty reads.

  • RepeatableRead: Ensures that data read multiple times during a transaction remains consistent, preventing non-repeatable reads.

  • Serializable: Provides the highest level of isolation by ensuring transactions are executed in a completely isolated manner, effectively serializing them.
